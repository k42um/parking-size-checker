# 🅿️ 駐車場 広さ判定 & 駐車シミュレーター

Google Map の衛星画像上で駐車場の **区画・通路・道幅・入口** を指定すると、
区画の大きさと道幅から **「狭め / ふつう / 広め」** を判定します。
さらに、入口から指定区画までの **駐車経路シミュレーション**（アニメーション付き）も行えます。

ブラウザだけで動く静的Webアプリです（ビルド不要）。

---

## 使い方

### 1. 起動

ローカルサーバー経由で開くのが確実です（`file://` 直開きでもおおむね動作します）。

```bash
cd parking-size-checker
python3 -m http.server 8000
# ブラウザで http://localhost:8000 を開く
```

### 2. APIキーの入力

画面上部に **Google Maps APIキー** を入力し「地図を表示」を押します。

> APIキーは [Google Cloud Console](https://console.cloud.google.com/) で取得します。
> 有効化が必要なAPI（いずれもお支払い設定が必要）:
> - `Maps JavaScript API` … 地図・描画・判定・シミュレーション
> - `Places API (New)` … 施設名・ランドマーク検索
>
> キーの「APIの制限」に上記2つを許可してください。不正利用防止のため
> 「HTTPリファラー制限」の設定も推奨します。

### 場所の検索・移動

画面上部の検索欄に **施設名・ランドマーク・住所** を入力すると、Google Maps と
同様に候補が表示され、選択するとその場所へ移動します
（Google Places API (New) / `PlaceAutocompleteElement` を使用）。

### 3. 各要素を描く

左パネルの描画モードを選び、衛星画像上に描画します。

| モード | 操作 | 用途 |
|--------|------|------|
| ▭ 区画を追加 | 区画の4隅をクリック→ダブルクリックで確定 | 区画の幅・奥行を計測（連続で複数追加可） |
| ― 通路の中心線 | 通路の中心に沿って折れ線を描く | 駐車経路のルートに使用 |
| ↔ 道幅を測る | 道を横切る線を2点で引く | 道幅（通路幅）を計測 |
| ⛳ 入口を置く | 入口位置をクリック | シミュレーションの出発点 |
| ✋ 操作（移動） | 通常の地図操作・図形の編集 | |

描画した図形は頂点をドラッグして微調整でき、計測値は自動で再計算されます。

### 4. 判定を見る

左パネルの「計測結果」「総合判定」に、区画幅・奥行・道幅と、それぞれの狭め/ふつう/広め、
および重み付き総合判定が表示されます。

### 5. 駐車シミュレーション

入口・通路中心線・区画をそろえたら、対象区画を選んで「経路を計算」→「▶ 再生」。
入口 → 通路 → 区画 への推奨経路を描画し、車アイコンが経路をたどります。
（地図上の区画をクリックしても対象として選択できます）

## 判定基準（普通車・日本の一般的な区画を想定）

| 項目 | 標準 | 狭め | 広め |
|------|------|------|------|
| 区画の幅 | 2.5 m | < 2.3 m | > 2.55 m |
| 区画の奥行 | 5.0 m | < 4.7 m | > 5.3 m |
| 道幅（通路） | 5.5〜6 m | < 5.0 m | > 6.0 m |

総合判定は「道幅 > 区画幅 > 奥行」の重みで加重平均しています。
基準値は [`app.js`](app.js) の `THRESHOLDS` で調整できます。

---

## ファイル構成

- `index.html` — 画面レイアウト
- `styles.css` — スタイル
- `app.js` — 地図・描画・計測・判定・シミュレーションのロジック

## 注意

- 計測値は衛星画像と手描きに基づく **概算** です。実測の代わりにはなりません。
- 距離計算は Google Maps の `geometry` ライブラリ（球面距離）と局所平面近似を併用しています。
